## 定义

**FFN（Feed-Forward Network，前馈神经网络）** 是 Transformer 架构中与注意力层并列的另一半核心组件。在每个 Transformer 层中，注意力机制负责"收集信息"（Token 之间的交互），FFN 负责"加工信息"（对每个 Token 独立进行非线性变换）。

FFN 的结构极其简洁：**两层全连接网络，中间夹一个激活函数**。第一层将输入扩展到更高维度（通常 4 倍），激活函数引入非线性，第二层再压缩回原始维度。它独立地、相同地作用于每一个 Token——不看上下文，只做"个人深加工"。

公式：`FFN(x) = W₂ · Activation(W₁ · x + b₁) + b₂`

在 LLM 中，FFN 占据了模型参数量的大部分（通常 2/3 以上），是模型"存储知识"的主要场所。


## 第一性

> 注意力负责"打听"（Token 之间互相看），FFN 负责"消化"（把打听到的信息深度加工后存进脑子里）。

## 比方

> 课堂上，讨论环节是注意力机制——同学之间互相交流、交换信息。讨论完之后，每个人回到座位上独立思考、整理笔记——这就是 FFN。同样一轮讨论，每个人消化理解的方式不同，形成的知识也不同。FFN 就是模型的"独立思考时间"。

## 英文术语拆解

| 英文全称 | Feed-Forward Network |
|----------|---------------------|
| 音标 | /fiːd ˈfɔːrwərd ˈnɛtwɜːrk/ |
| 缩写 | FFN |

**词源说明：**

- **Feed（喂入）**：将数据输入网络
- **Forward（向前）**：信息只沿一个方向传递（输入→输出），没有反馈回路
- **Network（网络）**：由多层节点组成的计算结构

"前馈"区别于"反馈"（Recurrent/Feedback）：信息单向流动，不会从输出倒流回输入。这是最基础的神经网络计算模式。

## 核心概念

### 位置无关性（Position-wise）
FFN 对序列中的每个 Token **独立且相同地**施加变换。无论 Token 在第几个位置，用的都是同一组权重 W₁、W₂。这意味着 FFN 不关心"这个词在句子中的位置"或"它和其他词的关系"——那些是注意力机制的工作。FFN 只做一件事：把输入的向量做一个固定的非线性变换。

### 扩展-压缩结构（Expansion-Contraction）
FFN 的经典结构是"先扩后缩"：

```
输入 (d_model) → [W₁ 扩展] → 中间层 (4 × d_model) → [激活函数] → [W₂ 压缩] → 输出 (d_model)
```

- **扩展层**：将 d_model 维向量映射到 4 倍维度（如 4096 → 16384），给模型更大的"思考空间"
- **激活函数**：引入非线性，让模型能学习复杂模式
- **压缩层**：将高维表示压回原始维度，供下一层使用

### 激活函数演进

| 激活函数 | 使用模型 | 特点 |
|---------|---------|------|
| ReLU | 早期 Transformer | 简单高效，但有"神经元死亡"问题 |
| GELU | BERT、GPT 系列 | 平滑的 ReLU 变体，梯度更稳定 |
| SwiGLU | LLaMA、Mistral、Qwen2 | 门控机制 + Swish 激活，当前主流选择 |
| GeGLU | 部分实验模型 | GELU 的门控版本 |

### FFN 是模型的"知识库"
研究表明，Transformer 的大部分事实知识和语言规则都存储在 FFN 的权重中。注意力机制负责"路由"——找到相关信息；FFN 负责"存储和计算"——把信息加工成有用的表示。

这也解释了 MoE（混合专家模型）的设计思路：既然 FFN 是知识的主要载体，那就把 FFN 拆成多个"专家"，每次只用其中几个，用更少的计算量获得更大的知识容量。

### FFN 变体

| 变体 | 说明 |
|------|------|
| 标准 FFN | 两层全连接 + 激活函数，原始 Transformer 设计 |
| SwiGLU FFN | 门控结构：`FFN(x) = (Swish(xW₁) ⊙ xW₃) · W₂`，LLaMA 系列标配 |
| MoE FFN | FFN 层替换为多个专家，推理时只激活部分（详见 MoE 词条） |
| Parallel FFN | FFN 与注意力并行计算，而非串行，减少延迟 |

## FFN 在 Transformer 层中的位置

```
输入
 └─► [LayerNorm]
 └─► Multi-Head Attention  ← "打听信息"
 └─► [残差连接 + LayerNorm]
 └─► FFN                   ← "消化信息"
 └─► [残差连接]
输出
```

每个 Transformer 层都包含一个注意力子层和一个 FFN 子层。典型 LLM 有几十到上百层这样的堆叠。

## 常见用法示例

### 示例：数值计算
假设 d_model = 4，FFN 中间维度 = 16（4 倍扩展）：

```
输入 x: [0.1, -0.3, 0.5, 0.2]  (维度 4)

第一层 W₁ (4×16):
  中间层 = GELU(x · W₁ + b₁)     → 维度变为 16

激活后: [0.4, 0.0, 0.8, ..., 0.1]  (维度 16)

第二层 W₂ (16×4):
  输出 = 中间层 · W₂ + b₂         → 维度变回 4

输出:   [0.2, -0.1, 0.6, 0.3]    (维度 4)
```

### FFN 参数量估算
以 LLaMA-7B 为例：
- d_model = 4096
- FFN 中间维度 = 11008（SwiGLU 有 3 个权重矩阵）
- 每层 FFN 参数 ≈ 3 × 4096 × 11008 ≈ 135M
- 32 层总 FFN 参数 ≈ 4.3B（占总参数 7B 的约 61%）

## 相关概念

- **MHA（多头注意力）**：Transformer 的另一半，负责 Token 间信息交互
- **MoE（混合专家模型）**：FFN 层的稀疏化扩展，推理时只激活部分专家
- **SwiGLU**：当前最主流的 FFN 激活函数变体
- **残差连接（Residual Connection）**：让信息"跳过"FFN 直接传递，解决深层训练问题
- **LayerNorm**：FFN 前的归一化层，稳定训练
- **Transformer**：FFN 所属的整体架构
