## 定义

智能体（Agent，全称 AI Agent）是一种能够自主感知环境、制定计划、调用工具并执行行动来实现特定目标的 AI 系统。与传统 AI 只能"你问我答"不同，智能体像一个能主动干活的助手：你给它一个目标，它自己拆解步骤、寻找信息、调用工具、检查结果，逐步完成任务。打个比方：传统 AI 像一本会说话的百科全书，而智能体像一个能帮你跑腿办事的私人助理。

## 第一性

> 一个能自主决策、调用工具、闭环执行目标的 AI 系统。

## 比方

> 传统 AI 是会说话的百科全书，你问它答；Agent 是能跑腿办事的私人助理，你给它一个目标，它自己拆解任务、找信息、调工具、检查结果，一步步把事办完。

### 组合理解

AI Agent = Artificial Intelligence（人工智能）+ Agent（行动者/代理）→ 具备人工智能、能代表用户自主行动的系统。"Agent"一词强调的是"能动性"——不仅知道什么，还能动手去做。

## 详细说明

### 核心特征

智能体区别于传统 AI 应用的关键特征包括：

- **自主性（Autonomy）**：无需人类逐步指令，能够独立决定"做什么"和"怎么做"。
- **反应性（Reactivity）**：能感知环境变化并实时调整行为。
- **主动性（Proactivity）**：不只是被动响应，而是主动规划和采取行动以实现目标。
- **社交性（Social Ability）**：能与其他智能体或人类协作交互。

### 核心架构

一个典型的智能体系统通常包含以下核心组件：

1. **大语言模型（LLM）**：作为"大脑"，负责理解意图、推理决策和生成内容。
2. **规划模块（Planning）**：将复杂目标拆解为可执行的子任务序列，并动态调整计划。
3. **记忆模块（Memory）**：短期记忆追踪当前任务进度，长期记忆保存历史交互和知识。
4. **工具调用（Tool Use）**：通过 API、数据库、搜索引擎等外部工具与真实世界交互。

### 工作流程

智能体的典型工作循环为：**感知 → 规划 → 行动 → 反馈 → 调整**，形成闭环。用户给定目标后，智能体自主完成以下流程：

1. 理解目标，将其分解为子任务
2. 选择合适的工具执行每个子任务
3. 根据执行结果评估进展
4. 如有必要调整计划，继续执行
5. 最终汇总结果交付给用户

### 使用场景

- 自动化软件研发（代码生成、调试、部署）
- 客户服务（自主处理复杂咨询、调用后台系统）
- 数据分析（自动查询、清洗、生成报告）
- 网页浏览与信息采集
- 多智能体协作完成复杂项目

## 相关概念

- **大语言模型（LLM）**：智能体的"大脑"，提供理解和推理能力。智能体是 LLM 的应用形态升级，从"对话"延伸到"行动"。
- **工具调用（Tool Use / Function Calling）**：智能体与外部世界交互的关键能力，使 LLM 不只是"说话"还能"做事"。
- **RAG（检索增强生成）**：智能体获取外部知识的常用手段，弥补 LLM 知识时效性不足的问题。
- **MCP（Model Context Protocol）**：一种标准化协议，让智能体能够统一地接入各种外部工具和数据源。
- **多智能体系统（Multi-Agent System）**：多个智能体协作完成复杂任务的架构，每个智能体承担不同角色。
