## 定义

**Discord** 是一款跨平台的即时通讯与数字社区平台，支持语音通话、视频通话、文字聊天、屏幕共享和文件传输等功能。用户通过创建或加入称为"服务器（Server）"的社区空间进行交流，每个服务器内可设置多个文字频道和语音频道，由管理员自定义权限与规则。

Discord 于 **2015 年 5 月**正式发布，由 Jason Citron 和 Stanislav Vishnevskiy 创立，最初面向在线游戏玩家提供低延迟语音聊天服务。凭借出色的音质、免费的基础功能和高度可定制性，Discord 迅速超越 TeamSpeak、Ventrilo 等传统游戏语音工具，成为游戏社区的首选通讯平台。

2020 年 COVID-19 疫情期间，Discord 的用户基础发生根本性转变——大量教育、创作、开源、Web3 等非游戏社区涌入，平台从"游戏语音工具"蜕变为**通用型社区协作平台**。截至 2025 年，Discord 月活跃用户超过 2.5 亿，注册用户逾 6.5 亿，日发送消息与语音分钟数超过 40 亿条。

## 英文术语拆解

| 英文全称 | Discord |
|----------|---------|
| 音标 | /ˈdɪskɔːrd/ |

**词源说明：**

"Discord" 在英语中意为**"不和谐、不一致"**，源自拉丁语 *discordia*（dis- "分离" + cor/cord- "心"），字面意思是"心意不合"。这个命名颇具反讽意味——一个旨在促进交流与连接的平台，却以"不和谐"命名。

据创始人 Jason Citron 回忆，团队在选名时尝试了大量与"声音""沟通""连接"相关的词汇，但均已被注册。最终选择 "Discord" 的原因是：这个词简短好记、辨识度高，且在游戏语境中隐含"打破不和谐、建立连接"的意味。

**关键术语对照：**

| 术语 | 含义 |
|------|------|
| Server（服务器） | 用户创建的社区空间，类似群组的顶层容器 |
| Channel（频道） | 服务器内的分类讨论区，分为文字频道和语音频道 |
| Nitro | Discord 的付费订阅服务，提供增强功能 |
| Bot（机器人） | 自动化程序，可执行管理、娱乐、音乐等任务 |
| Guild | 早期 API 中对服务器的称呼，现仍用于开发者文档 |

## 核心功能/特性

### 服务器（Server）体系
用户可创建免费的服务器作为社区主空间。每个服务器支持最多 500 个频道（含文字和语音），管理员可设置精细的角色权限系统（Role-based Permissions），控制成员的读写、发言、管理等权限。

### 实时语音与视频
Discord 采用自研的语音引擎，提供低延迟（通常低于 40ms）的高质量语音通话。支持语音频道"随时进出"的模式——用户进入频道即开始通话，离开即结束，无需拨号等待。视频通话和屏幕共享（Go Live）功能允许用户向频道或好友直播自己的屏幕画面。

### 文字频道与线程（Threads）
文字频道支持富文本消息（Markdown 格式）、图片/视频/文件分享、@提及、嵌入（Embed）、回复引用等。线程功能允许在频道内创建子讨论串，保持话题的条理性。2022 年新增的论坛频道（Forum Channel）进一步提供了结构化的话题讨论形式。

### 机器人（Bot）生态
Discord 拥有成熟的 Bot 开发框架和 API，支持多种编程语言接入。社区已积累超过 1200 万个活跃 Bot，功能涵盖自动管理（如 AutoMod）、音乐播放、游戏数据查询、AI 助手（如 Clyde）、投票、欢迎新人等场景。

### Nitro 订阅服务
Discord Nitro 是平台的主要收入来源，分为个人版和基础版两档。付费用户可享受更大文件上传限制、高清视频流、自定义标签（Tag）、动态头像、全局自定义表情使用、服务器提升（Server Boost）等增值功能。

### 端到端安全与隐私
Discord 对文字、语音、视频通信实施加密传输。平台提供两步验证（2FA）、安全密钥、登录通知等安全机制，以及内容审核工具和自动有害链接检测。

### 跨平台支持
Discord 提供 Windows、macOS、Linux 桌面客户端，iOS 和 Android 移动客户端，以及 Web 浏览器版本，用户数据通过云端实时同步。

### 活动（Activities）与阶段频道（Stage）
Activities 功能允许用户在语音频道内一起玩小游戏、进行互动活动。Stage 频道则提供类似"演讲厅"的模式，支持主持人和听众的分级互动，适用于 AMA（Ask Me Anything）、讲座、播客录制等场景。

## 常见用法示例

```text
# 典型使用场景

1. 游戏开黑：组队玩家在语音频道实时沟通，配合游戏战术
2. 开源社区：开发者在文字频道讨论技术方案，用 Bot 管理 Issue 和 PR
3. 在线教育：教师开设 Stage 频道进行远程授课，学生在文字频道提问
4. 创作者粉丝群：YouTuber 或主播建立粉丝社区，发布动态、互动聊天
5. DAO/Web3 社区：去中心化组织通过 Discord 进行治理讨论和公告发布
6. 企业小团队协作：初创团队用服务器替代 Slack 进行内部沟通（免费方案更具吸引力）
```

```text
# 服务器结构示例

我的游戏社区（Server）
  ├── #公告（文字频道 - 仅管理员可发）
  ├── #闲聊（文字频道 - 所有人可发）
  ├── #游戏攻略（文字频道 + 线程）
  ├── 语音大厅（语音频道 - 随时进出）
  ├── 组队开黑（语音频道）
  └── 直播舞台（Stage 频道 - 演讲模式）
```

## 相关概念

- **TeamSpeak**：老牌游戏语音通讯软件，以低延迟著称，曾是 Discord 的主要竞争对手
- **Slack**：面向企业团队的即时通讯与协作平台，功能更接近办公场景
- **VoIP（Voice over IP）**：通过互联网协议传输语音通信的技术，Discord 的核心底层技术之一
- **WebRTC**：开源的实时通信技术，Discord 的语音/视频功能部分基于此技术构建
- **Serverless Bot**：基于云函数运行的 Discord Bot，无需维护服务器即可部署自动化功能
- **社区运营（Community Management）**：围绕在线社区进行成员管理、内容运营和文化建设的实践
- **Nitro/Guild ID**：Discord 生态中的核心概念，分别代表付费订阅和社区唯一标识
