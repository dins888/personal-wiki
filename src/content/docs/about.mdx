# 关于这个百科

## 项目目标

这是一个个人百科知识库，用于系统化地沉淀日常工作和生活中遇到的新概念、新术语。

## 查证工作流

每个词条的诞生都经过以下流程：

### 完整通道（新概念/有争议概念）

1. **摄入**：用户提出概念、截图或文档
2. **检索**：按信源权威等级（T1-T5）分层检索
3. **审核**：三角验证法，至少 3 个独立权威来源
4. **确认**：呈现给用户做最终判断
5. **沉淀**：写入百科，标注版本号和信源

### 快速通道（常见高频概念）

对于已经在权威来源中有明确一致定义的高频概念（如 CLI、API、DNS），1 个 T1/T2 来源即可确认，跳过三角验证，节省时间。

## 信源权威等级

| 等级 | 来源类型 | 举例 |
|------|---------|------|
| **T1** | 官方文档/标准/规范 | RFC、W3C、ISO、IEEE |
| **T2** | 权威百科/教科书 | 大英百科全书、学术教科书 |
| **T3** | 专业媒体/行业报告 | 行业白皮书、专业期刊 |
| **T4** | 社区共识 | Stack Overflow 高票、Reddit 高赞 |
| **T5** | 个人博客/自媒体 | 仅作补充参考 |

## 词条规范

每个词条包含：
- **定义**：简洁准确的核心定义
- **英文术语拆解**：全称 + IPA 音标 + 逐词解释
- **信源列表**：标注每个来源的权威等级
- **变更记录**：版本号 + 变更日期 + 变更内容

## 技术架构

- 生成框架：Astro + Starlight
- 托管：GitHub Pages
- 版本管理：Git
